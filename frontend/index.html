About

Problem statement

Given a single broadcast-view still image of a football match, predict the (x, y) pitch position in meters of every player visible in the image. The pitch is a 105 × 68 m field with the origin at the pitch center; +X points to the right goal and +Y points to the bottom touchline. Models receive only the image and must return the same set of coordinates as the ground truth.

Dataset

385 broadcast-view images sampled from the SoccerNet Game State Reconstruction (SN-GSR) test + valid splits. For each image the ground truth lists the in-camera players' pitch-meter positions — not all 22, only those visible in the frame.

Scoring algorithm

Each image's score is GOSPA (with $c = 30$ m, $p = 1$, $\alpha = 2$) divided by the ground-truth player count:

$$ \text{score}_i \;=\; \frac{\text{GOSPA}\!\left(\hat{X}_i,\, X_i;\; c=30,\, p=1,\, \alpha=2\right)}{\max(n_{\text{gt},\,i},\, 1)} $$

The headline number is the average score over the entire dataset, in meters. Lower is better.

Leaderboard

Mean per-image normalized GOSPA over the 385-image dataset, in meters. Lower is better.

Architecture grouping

Models are colored by how their vision pipeline handles broadcast-resolution input — the architectural choice most likely to affect player-coordinate accuracy on this task.

  • Native dynamic resolution — the vision encoder accepts the full image in one pass at its real aspect ratio, so spatial relationships across the pitch survive into the language model.
  • Early fusion — no separate vision encoder; image patches and text tokens flow through one unified transformer trained jointly on both modalities from scratch.
  • Tiled / Pan-and-Scan — fixed-square vision encoder; the image is sliced into tiles encoded independently, leaving cross-tile spatial reasoning to the language model.
  • Closed / undisclosed — proprietary models whose vision pipeline is not publicly documented.
  • CV baseline — classical computer-vision pipeline, not an LLM.

Contribute

Submitting a new model or improvement to the benchmark.

Submitting a new model

  1. Run the harness against the manifest (manifest.csv) to produce a per-image score CSV with columns image_id, gospa_normalized.
  2. Place the CSV at llm_results/<model-name>_scores.csv.
  3. Add the model to the SOURCES list in frontend/app.js, plus an entry in PARAMS (parameter count) and BUCKETS (vision-pipeline architecture).
  4. Open a pull request describing the model, the prompt used, and any deviations from the standard harness.

Reporting issues

For bugs, unclear documentation, or questions about scoring, please open a GitHub issue.

Reproducing baseline runs

The CV baseline and all 10 LLM runs are independently reproducible from the repo. See the top-level README for environment setup and run commands.
